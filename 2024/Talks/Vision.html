I'm here because I. I'm only in front of you now because the Vision is a really good way to frame all the work we collectively do at W3C, and it's a good way to put all the talks here at the AC meeting into a common context. Although the Vision document is a work started by the AB to describe W3C, the goal we have is to unite the mission we are all on, by representing shared values of the W3C community, which in turn will guide the community’s work.

Why do we need a Vision?

  • We need principles to help guide us without a Director
  • Growing privacy and other ethical concerns on the web
  • Continued growth and importance of the Web in the world around us
  • We must improve the integrity and trustworthiness of the Web

It is particularly important at this point in W3C's history to have a shared Vision statement - partly because make up for no longer having a singular Director, by building a set of modern grounded community values, to help consistently guide W3C's decisions, but also because there have been growing problems on the web - privacy and other ethical concerns, as well as unprecedented growth in the importance of the Web to the world around us.

We feel we must improve the integrity and trustworthiness of the web, because the credibility for W3C's future needs to come from our community consensus.

Vision for the Web

  • The Web is for all humanity.
  • The Web is designed for the good of its users.
  • The Web must be safe for its users.
  • There is one interoperable world-wide Web.

Before we get into the vision of what W3C is, we needed to sketch out what we believe underlies the Web itself; this is clearly a space filled by the TAG’s Ethical Web Principles, which you’ll hear more about later today from Dan; these basic points come from that document: The web is for all Humanity. The web is designed for the good of its users, and it must be safe for those users; finally, there is one single interoperable worldwide web. This is a great groundwork for the Web...

Vision for W3C

W3C leads the community in defining a World Wide Web that puts users first, by developing technical standards and guidelines to empower an equitable, informed, and interconnected society.

The fundamental function of W3C today is to provide an open forum where diverse voices from around the world and from different organizations and industries work together to evolve the web by building consensus on voluntary global standards for Web technologies.

... and what we needed to do from there was articulate the vision for w3c itself: that W3C is leading the community to define a web that puts users first, by developing standards and guidelines to empower an equitable informed and interconnected society.

The fundamental function of w3c is really to provide an open forum where a whole lot of diverse voices from around the world different organizations different Industries can work together to evolve the web by building consensus on our voluntary web standards

Principles/Values

  • User-first
  • Multi-stakeholder
  • Diversity
  • Thorough Review
  • Consensus
  • Free to Implement
  • Open Participation
  • Interoperability
  • Incubation
  • Avoid Centralization
  • Collaboration

Now the vision document goes into a lot more detail into specific principles and values that we need to apply and to some degree how we need to apply this. I don't want to go into each one of those here in detail; I want you to go read the Vision instead. It should be no suprise that we are user first, that we're multi-stakeholder, that diversity is important to us, that we need to work to avoid centralization. These principles should run through all of the work that we do together at W3C; if it doesn't, we probably don't have the vision quite right yet.
